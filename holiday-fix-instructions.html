CheckLoops Holiday Data Fix

This page provides instructions for fixing the holiday data display issue in the admin dashboard.

Step 1: Fix Database Values

First, we need to update the holiday_taken and approved_holidays_used values in the database:

-- SQL Fix for holiday_taken and approved_holidays_used columns
WITH holiday_totals AS (
  SELECT 
    hr.user_id,
    SUM(hr.total_days) as total_days
  FROM 
    "4_holiday_requests" hr
  WHERE 
    hr.status = 'approved'
  GROUP BY 
    hr.user_id
)
UPDATE 
  master_users mu
SET 
  holiday_taken = COALESCE(ht.total_days, 0),
  approved_holidays_used = COALESCE(ht.total_days, 0)
FROM 
  holiday_totals ht
WHERE 
  mu.auth_user_id = ht.user_id;

You can run this SQL directly in the Supabase dashboard SQL editor.

Step 2: Add Fix to Admin Dashboard

Add the following script tag to your admin-dashboard.html file right before the closing </body> tag:

<script src="admin-dashboard-holiday-fix.js"></script>

Step 3: Browser Console Method (Quick Fix)

If you don't want to modify the HTML file, you can apply the fix temporarily using the browser console:

  1. Open the admin dashboard in your browser
  2. Open the developer console (F12 or right-click > Inspect > Console)
  3. Copy and paste the following code:

Notes About the Fix

The fix does the following:

  • Ensures the holiday_taken and approved_holidays_used columns in the database have the correct values
  • Overrides the loadStaffEntitlementCards function to fetch the latest data directly from the database
  • Adds a refresh button to the holiday management view to reload data
  • Formats time values correctly in the display
  • Handles both GP and non-GP staff display formats properly

Issue Summary

The issue occurred because:

  • The holiday_taken and approved_holidays_used columns in the master_users table weren't being updated when holiday requests were approved
  • The admin dashboard was showing the outdated values from these columns
  • The SQL fix updates these columns with the correct totals from approved holiday requests
  • The JavaScript fix ensures the UI displays the latest values from the database
